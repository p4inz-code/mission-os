# Security Policy

## Our Commitment

Security is a core principle of Mission OS.

We appreciate responsible disclosure and will investigate all legitimate security reports promptly and respectfully.

Please do not disclose security vulnerabilities publicly until they have been reviewed and, where appropriate, resolved.

---

## Reporting a Vulnerability

If you discover a potential security issue:

- Do not open a public GitHub issue.
- Contact the project using the security contact information provided in `SECURITY_CONTACT.md`.
- Include as much detail as possible to help reproduce the issue.

Helpful information includes:

- A description of the vulnerability
- Steps to reproduce
- Affected versions or commits
- Potential impact
- Proof of concept (if safe to share)
- Suggested mitigation (optional)

---

## Response Process

Every report is reviewed according to the following process:

1. Acknowledge receipt.
2. Validate the report.
3. Assess severity and impact.
4. Develop and test a fix.
5. Publish the fix when ready.
6. Credit the reporter where appropriate.

---

## Supported Versions

Supported releases are listed in `SUPPORTED_VERSIONS.md`.

Development builds may contain incomplete features and should not be considered production-ready.

---

## Scope

This policy applies to:

- Mission OS source code
- Official releases
- Build scripts
- Documentation that affects security
- Project infrastructure

---

## Thank You

Responsible disclosure helps make Mission OS more secure for everyone.

Thank you for helping improve the project.
