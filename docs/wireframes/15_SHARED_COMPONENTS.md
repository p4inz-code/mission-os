# Purpose

Defines the shared component anatomy used consistently across every Mission OS application.

This document serves as the implementation reference for reusable interface components.

---

# Navigation Components

- Top Panel
- Sidebar
- Dock
- Toolbar
- Breadcrumb
- Tabs
- Context Menu
- Status Bar

---

# Form Components

- Text Field
- Password Field
- Search Field
- Number Field
- Dropdown
- Checkbox
- Radio Button
- Toggle Switch
- Date Picker
- File Picker

---

# Buttons

- Primary
- Secondary
- Tertiary
- Destructive
- Icon
- Split Button
- Floating Action Button

All button variants follow the Design System.

---

# Cards

Shared card types:

- Information
- Statistics
- Dashboard
- Application
- Device
- Recommendation
- Warning
- Security

---

# Lists

- Standard List
- Compact List
- Tree View
- Grid View
- Table View

---

# Dialogs

Supported dialogs:

- Confirmation
- Error
- Success
- Authentication
- Progress
- Warning

Nested dialogs are prohibited.

---

# Feedback

Standard feedback components:

- Toast
- Alert
- Banner
- Snackbar
- Progress Bar
- Progress Ring
- Skeleton Loader

---

# Empty States

Every empty state includes:

- Icon or Illustration
- Title
- Description
- Primary Action
- Optional Secondary Action

---

# Loading States

Loading interfaces should prefer:

- Skeleton screens
- Progressive loading
- Incremental rendering

Blocking full-screen spinners should only be used when unavoidable.

---

# Error States

Every error component provides:

- Human-readable explanation
- Recovery action
- Optional diagnostics link

---

# Accessibility

Every shared component supports:

- Keyboard navigation
- Screen readers
- High contrast
- Reduced motion
- Scalable typography

---

# Acceptance Criteria

Shared components must behave identically across all Mission OS applications and conform to the Design System and Master UX Specification.
