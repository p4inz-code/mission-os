# Mission OS Crash Reporting

**Document ID:** MOS-ENG-008

## Purpose

Defines crash collection and recovery.

---

# Workflow

Crash Detected

↓

Save Memory Information

↓

Write Crash Log

↓

Offer Recovery

↓

Restart Component

↓

Notify User

---

# Rules

- Crash reports remain local by default.
- User controls sharing.
- Sensitive information is removed.

---

# Recovery

Support:

- Safe Mode
- Restart Component
- Rollback
- Diagnostics

---

# Definition of Done

Crashes are recoverable with useful diagnostic information.
