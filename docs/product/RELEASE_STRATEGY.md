# Release Strategy

## Development

- Continuous development
- Frequent internal builds

---

## Alpha

Purpose

- Core functionality
- Developer testing

---

## Beta

Purpose

- Feature validation
- Wider hardware testing

---

## Release Candidate

Purpose

- Final bug fixing
- Stability verification

---

## Stable

Requirements

- QA complete
- Security review passed
- Performance review passed
- Documentation complete

---

## Hotfix

Reserved for

- Critical security issues
- Critical regressions
- Data-loss bugs

---

## Support Policy

Development → No guarantees

Alpha → Experimental

Beta → Best effort

Stable → Supported

LTS → Long-term maintenance
