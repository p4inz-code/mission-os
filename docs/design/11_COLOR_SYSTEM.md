# Mission OS Color System

**Document ID:** MOS-DESIGN-011

**Version:** 1.0

---

# 1. Purpose

Defines the official color system used throughout Mission OS.

---

# 2. Principles

Colors must:

- Communicate meaning
- Maintain accessibility
- Support light and dark themes
- Remain consistent

---

# 3. Color Categories

Mission OS uses:

- Primary
- Secondary
- Surface
- Background
- Border
- Text
- Success
- Warning
- Error
- Information

---

# 4. Semantic Colors

Success

- completed
- healthy
- verified

Warning

- attention
- pending
- caution

Error

- failed
- blocked
- destructive

Information

- neutral
- guidance
- status

---

# 5. Theme Support

Every semantic color has:

- Light Theme variant
- Dark Theme variant
- High Contrast variant

---

# 6. Usage Rules

Colors must never be used as the only indicator of state.

Icons, labels or patterns must reinforce meaning.

---

# 7. Accessibility

Minimum WCAG AA contrast.

Critical interfaces target WCAG AAA where practical.

---

# 8. Acceptance Criteria

The color system remains consistent across every Mission OS application.
