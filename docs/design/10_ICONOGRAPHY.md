# Mission OS Iconography

**Document ID:** MOS-DESIGN-010

**Version:** 1.0

---

# 1. Purpose

Defines the icon language used across Mission OS.

---

# 2. Principles

Icons must be:

- Simple
- Recognizable
- Consistent
- Scalable
- Accessible

---

# 3. Style

Official icons use:

- Rounded geometry
- Uniform stroke weight
- Minimal detail
- Optical balance

---

# 4. Categories

Icon groups include:

- System
- Navigation
- Files
- Devices
- Network
- Security
- Privacy
- Recovery
- Applications
- Status

---

# 5. States

Icons support:

- Default
- Active
- Disabled
- Selected
- Warning
- Error

---

# 6. Sizes

Standard sizes:

- 16 px
- 20 px
- 24 px
- 32 px
- 48 px
- 64 px
- 128 px

---

# 7. Color Rules

Icons inherit semantic colors from the design system.

Status colors are reserved for:

- Success
- Warning
- Error
- Information

---

# 8. Accessibility

Icons that represent actions or information must include accessible labels.

Icons alone should never be the only means of conveying critical information.

---

# 9. Acceptance Criteria

All official applications use the shared icon library with no custom icon styles.
